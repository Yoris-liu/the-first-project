## 📖 项目简介 | Project Overview

**中文**

这是一个初学者的全栈项目，构成了一个电商平台后台管理系统的完整基础框架。项目采用前后端分离架构，前端使用 Vue 3 + Element Plus 构建单页应用（SPA），后端使用 Spring MVC 4.3 + MyBatis 3.4 提供 RESTful API，数据库使用 MySQL。

项目涵盖了电商后台常见的功能模块：店铺管理、商品管理（含分类管理）、订单管理、用户管理、权限角色管理、操作日志记录、文件上传以及数据可视化等。

> ⚠️ **数据库声明**：本项目数据库（`eshopDB03.sql`）中的表结构和测试数据由 AI 生成，仅用于项目的功能演示和开发测试，**不包含任何真实商业数据**。如数据中存在任何与现实企业、品牌或产品的雷同，纯属巧合。

**English**

This is a beginner's full-stack project that forms the basic framework of an e-commerce backend management system. It uses a front-end/back-end separation architecture: the frontend is a Vue 3 + Element Plus SPA, the backend provides RESTful APIs via Spring MVC 4.3 + MyBatis 3.4, and data is stored in MySQL.

The project covers common e-commerce admin modules: shop management, product management (with categories), order management, user management, role/permission management, operation logging, file upload, and data visualization.

> ⚠️ **Database Disclaimer**: The database (`eshopDB03.sql`) in this project — including all table structures and test data — was generated by AI. It is intended solely for feature demonstration and development testing purposes and **does not contain any real business data**. Any resemblance to real enterprises, brands, or products is purely coincidental.

### 后端分层架构 | Backend Layered Architecture

```
┌──────────────────────────────────────────────────┐
│               Controller 层 (控制器)                │
│     接收 HTTP 请求，调用 Service，返回 JSON 响应      │
├──────────────────────────────────────────────────┤
│              Service 层 (业务逻辑)                   │
│    接口 + 实现，处理业务逻辑，调用 Mapper 访问数据     │
├──────────────────────────────────────────────────┤
│          Dao/Mapper 层 (数据访问)                    │
│    MyBatis 接口 + XML 映射，执行 SQL 操作数据库      │
├──────────────────────────────────────────────────┤
│               POJO 层 (实体模型)                     │
│        数据库表映射的 Java 对象（Lombok @Data）       │
└──────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────┐
│  Common 公共模块                                    │
│  Result（统一响应）│ JWTUtil（令牌）│ LogAspect（AOP日志）│
│  LoginInterceptor（拦截器）│ GlobalExceptionHandler  │
└──────────────────────────────────────────────────┘
```

1. 启动 MySQL 服务
2. 创建数据库：

```sql
CREATE DATABASE eshopDB03 CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. 导入数据：

```bash
# 在 MySQL 命令行或图形化工具（如 Navicat）中执行
mysql -u root -p eshopDB03 < eshopDB03\(2026-03-25\).sql
```

4. 修改数据库连接配置（可选）：

编辑 `eshop2403/src/main/resources/database.properties`，修改用户名和密码：

```properties
jdbc.username=你的数据库用户名
jdbc.password=你的数据库密码
```

### 第 3 步：启动后端 | Step 3: Start Backend

**方式一：使用 IntelliJ IDEA**

1. 使用 IntelliJ IDEA 打开 `eshop2403/` 目录
2. 配置 Tomcat：
   - Run → Edit Configurations → Add New Configuration → Tomcat Server → Local
   - 选择 Tomcat 安装目录
   - Deployment → Add → Artifact → `eshop2403:war exploded`
   - Application context: `/eshop2403`
   - Server → Port: `90`（可选，默认为 8080。注意前端 axios baseURL 配置为 `http://127.0.0.1:90/eshop2403`，如果修改端口需同步修改前端配置）
3. 点击 Run 启动

**方式二：使用 Maven 命令行打包部署**

```bash
cd eshop2403
mvn clean package
# 将 target/eshop2403.war 复制到 Tomcat 的 webapps 目录
cp target/eshop2403.war ${TOMCAT_HOME}/webapps/
# 启动 Tomcat
${TOMCAT_HOME}/bin/startup.sh
```

后端默认运行在：`http://127.0.0.1:90/eshop2403`

### 第 4 步：启动前端 | Step 4: Start Frontend

```bash
cd eshop
npm install
npm run dev
```

前端默认运行在：`http://localhost:5173`

## 📦 部署指南 | Deployment

### 生产环境部署 | Production Deployment

#### 前端构建 | Frontend Build

```bash
cd eshop
npm install
npm run build
# 构建产物在 dist/ 目录
```

将 `dist/` 目录下的文件部署到 Nginx 或任何静态文件服务器。

##### Nginx 配置示例 | Nginx Config Example

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端 SPA
    location / {
        root /path/to/eshop/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /eshop2403/ {
        proxy_pass http://127.0.0.1:90;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

> ⚠️ 生产环境部署时，修改 `eshop/src/utils/request.js` 中的 `baseURL` 为实际的服务器地址。

#### 后端部署 | Backend Deployment

```bash
cd eshop2403
mvn clean package -DskipTests
# 将 target/eshop2403.war 部署到 Tomcat
```
